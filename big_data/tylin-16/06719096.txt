It described a model of a semantic index that can be incrementally composed by a combination of base extractor policies, knowledge bases, and entailments. A model for querying semantic index was discussed that allows both searching for matching documents and/or matching triples. An experimental study involving TREC documents demonstrated the feasibility of our approach The proposed semantic indexing framework is intended to create a synergy between the database, semantic web, and information extraction communities. Specifically, the integration and inference benefits of semantic technologies become available for use against unstructured data in the databases. The semantic web community benefits as large volumes of RDF data, hitherto residing as unstructured text can now drive the development of semantic applications Information extraction community benefits as their technologies are put to immediate use by the database and semantic web communities Although extracted data, stored in standard RDF format, is amenable to querying, it tends to be verbose. Further work is needed, in conjunction with the information extraction community, to come up with levels of summarization that will also result in compact and efficient SPARQL query formulation.
